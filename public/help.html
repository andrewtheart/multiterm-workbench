MultiTerm Workbench — Help

A local workbench for running and arranging many PowerShell sessions in one window.

Getting started

  • Click Terminal (top-right) or press Ctrl+Shift+T to open a new session.
  • Pick the shell (PowerShell 7, Windows PowerShell, Command Prompt, or WSL) from the Shell dropdown before creating a session.
  • Set a starting folder in the CWD box to launch new sessions there.
  • Type in any pane just like a normal terminal — Ctrl+C, Tab completion, and PSReadLine editing are forwarded through a real pseudo-terminal.

Arranging terminals

Use the layout controls panel on the left to change how panes are tiled:

  • Auto fit — panes size themselves to fill the workbench.
  • Fixed columns / rows — lock the grid to a set number of columns or rows.
  • Horizontal strip / Vertical stack — line panes up in a single row or column.
  • Focus rail — enlarge the active pane while keeping the rest reachable.
  • Manual canvas — drag panes by their header and resize from the lower-right corner.

Drag any terminal by its header to the top, bottom, left, or right edge to snap it there; the remaining panes reflow to fill the space.

Click the minimize (−) button in a pane’s header to tuck it away to the status bar without closing the session. The pane hides and the others reflow; a chip appears at the bottom — click it to restore the pane right where it was.

Working across sessions

  • Broadcast (Ctrl+Shift+B) sends one command to every terminal, or to a chosen scope.
  • Sync input mirrors your typing into all panes at once — toggle it in the layout panel.
  • The top Search box filters panes by their on-screen text; non-matching panes stay hidden until they match again or you clear the search.
  • The bottom-left trash button closes every pane and asks the bridge to stop all running sessions.

Diagnostics & logs

Click the chevron in the bottom-right corner to open the log console — a live, tailed view of everything the app and the local bridge are doing (connections, session start/exit, broadcasts, workspace changes, and any errors). Use the minimum level selector to filter to warnings or errors, copy the visible log to the clipboard, or clear it. A dot on the chevron flags a new error while the console is closed.

Workspaces & sessions

  • Save the current set of panes and layout as a named workspace, then restore it later from the layout panel.
  • Enable Restore last session to reopen your terminals automatically the next time you launch MultiTerm.

Tips & troubleshooting

MultiTerm runs a small local bridge that owns the real PowerShell processes. It listens only on 127.0.0.1, so sessions are reachable from this machine alone.

  • To run on a different port, launch the bridge with a PORT value, e.g. PORT=4000.
  • If a pane says the bridge is offline, the local bridge stopped — relaunch MultiTerm from the Start Menu or shortcut.
  • Right-click inside a pane for copy, paste, find, clear, split, and restart actions.
